# Project Audit: Humantic AI

## 1. What This App Actually Does

Humantic AI is an AI-powered research companion that allows users to:
- Sign up/login via Supabase authentication
- Complete onboarding to set research preferences (domain, knowledge gap, depth preference)
- Submit research topics which trigger background AI research tasks via Celery
- Chat with an AI assistant (Google Gemini) that can detect research intent and auto-create research topics
- View research findings categorized by type (deep_insight, trend, opportunity, experimental)
- Approve or dismiss findings
- Pin interests for continuous monitoring (CRUD only, no actual monitoring implemented)
- Receive real-time updates via WebSocket when new findings/topics are created

The research process is entirely LLM-based (Gemini 2.0-flash) - no web scraping or external data sources are integrated. The system generates research questions, performs "research" via the LLM's knowledge, and synthesizes findings.

## 2. Tech Stack

### Backend
- **FastAPI** - Web framework
- **Supabase** - Database (PostgreSQL) and Authentication
- **Google Generative AI** - Gemini 2.0-flash and 2.5-flash for AI processing
- **Celery** - Background task queue
- **Redis** - Celery broker and result backend
- **Pydantic v2** - Data validation
- **Uvicorn** - ASGI server
- **WebSockets** - Real-time communication

### Frontend
- **React 19** - UI framework
- **Vite 8** - Build tool
- **React Router DOM 7** - Routing
- **Axios** - HTTP client
- **Framer Motion 12** - Animations
- **React Markdown** - Markdown rendering

## 3. Fully Working End-to-End

- **Authentication**: Signup, login, logout via Supabase auth
- **Onboarding**: 3-step wizard saves user profile (domain, knowledge_gap, depth_preference) to Supabase
- **Research Submission**: POST /api/research creates topic record, triggers Celery task
- **Chat System**: Create sessions, send messages, receive AI responses from Gemini, auto-detect research intent
- **Findings Management**: List findings, filter by category, approve/dismiss status updates
- **Pinned Interests**: Create, list, delete (soft delete via is_active flag)
- **WebSocket**: Real-time push of new findings and topics to connected clients
- **Interaction Logging**: All user actions logged to interaction_logs table

## 4. Frontend-Only with No Backend Wired Up

None. All frontend components are wired to real API endpoints. The `mockData.js` file exists but is not used in any component - all data comes from the backend.

## 6. API Endpoints

### Authentication (`/api/auth`)
- `POST /api/auth/signup` - Create user account
- `POST /api/auth/login` - Authenticate user
- `POST /api/auth/logout` - End session

### Onboarding (`/api`)
- `POST /api/onboarding` - Complete onboarding, save user profile
- `GET /api/onboarding/test` - Test protected route

### Research (`/api`)
- `POST /api/research` - Create research topic, trigger Celery task
- `GET /api/research` - List user's research topics

### Findings (`/api`)
- `GET /api/findings` - List findings (optional filters: topic_id, category)
- `GET /api/findings/{finding_id}` - Get single finding
- `PATCH /api/findings/{finding_id}` - Update finding status (approved/dismissed)

### Pins (`/api`)
- `POST /api/pins` - Create pinned interest
- `GET /api/pins` - List active pinned interests
- `DELETE /api/pins/{pin_id}` - Soft delete pinned interest

### Chat (`/api`)
- `GET /api/sessions` - List chat sessions
- `POST /api/sessions` - Create new chat session
- `GET /api/sessions/{session_id}/messages` - Get session messages
- `POST /api/sessions/{session_id}/messages` - Send message, get AI response

### WebSocket
- `WS /ws/findings?token={jwt}` - Real-time findings/topics updates

### Root
- `GET /` - Health check endpoint

## 7. What a User Can Actually Do Locally Today

**Prerequisites to run:**
- Supabase project with required tables (users, research_topics, findings, pinned_interests, chat_sessions, chat_messages, interaction_logs)
- Redis server running on localhost:6379
- Google Gemini API key
- Environment variables configured (SUPABASE_URL, SUPABASE_ANON_KEY, SUPABASE_SERVICE_ROLE_KEY, GEMINI_API_KEY)

**If all prerequisites are met, a user can:**
1. Navigate to landing page, sign up for account
2. Complete 3-step onboarding (current work, knowledge gap, depth preference)
3. Submit research topics via dashboard input
4. Switch to chat view and converse with AI assistant
5. View research findings as they appear (via WebSocket or refresh)
6. Approve or dismiss findings
7. Add/remove pinned interests
8. See real-time updates when new findings are generated

**What will NOT work:**
- Scheduled background tasks (missing implementation)
- Actual monitoring of pinned interests (no implementation)
- Email verification (not configured)
- Password reset (not implemented)
- Any web search or external data fetching (not implemented)

**Current state:** The app is a functional MVP with authentication, basic research submission, chat, and findings management. The research is purely LLM-generated without external data sources. Several scheduled features are defined but not implemented.
